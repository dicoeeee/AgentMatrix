AgentMatrix 功能流程

把 AI 编码后的交付过程，变成可验证、可恢复、可追踪的工程工作流

当前 MVP 聚焦 MR 预检：从静态检查、测试检查、代码审查，到 MR 标题和描述生成，全过程以结构化证据和运行状态驱动，而不是依赖临时提示词和人工记忆。

核心价值

AgentMatrix 解决的是“AI 完成代码后，如何可信地走完交付前检查”的流程管理问题。

4 MR 预检主阶段：静态检查、测试检查、代码审查、MR 准备。
2 每阶段最少两类证据：执行证据与验证器证据。
1 项目本地 .agentmatrix 目录承载工作流、状态、产物和可视化。
0 MVP 不做自动 push、建 PR/MR、分配 reviewer 或监听 CI。

MR Preflight 四阶段

每个阶段都有明确输入、输出、完成条件、修复策略和 verifier 角色。

阶段 01static_check

静态检查

发现并执行 lint、typecheck、format check、audit 等项目已有静态门禁。

  • 读检查可并行跑在隔离工作区。
  • 安全机械修复可串行执行。
  • 输出 stage-report 和语言参考信息。
阶段 02test_check

测试检查

基于 package scripts 或 Make target 发现并执行测试命令。

  • 依赖 static_check 报告。
  • 在隔离工作区运行，避免污染源码。
  • 阻止 snapshot、golden、fixture 更新类命令。
阶段 03code_review

代码审查

消费静态和测试证据，按多个风险视角汇总可行动发现。

  • 覆盖正确性、安全、可维护性、性能、数据、API。
  • 合并重复发现，按根因归类。
  • 缺少前序证据时阻断推进。
阶段 04mr_prepare

MR 准备

把前三阶段报告整理成 MR 标题、描述和验证摘要。

  • 生成 title.md 和 description.md。
  • 说明变更、验证结果、发现和注意事项。
  • 不执行 push、建 PR/MR 或 CI 监听。

CLI 与 Agent 框架的关系

AgentMatrix 不替代 OpenCode 这类 agent 框架，而是在其上方提供稳定的流程契约和状态控制。

AgentMatrix CLI：控制平面

负责定义、启动、恢复和验证工作流，是 run state 的权威来源。

  • init：安装 workflow、技能模板和可选平台 agent 模板。
  • run / resume：通过 runtime adapter 执行或恢复工作流。
  • driver：向交互式 agent 框架输出 Stage Invocation JSON。
  • status / visualize：读取状态、证据和 trace，展示执行进度。
⇄

OpenCode：执行平面

负责调用主 driver agent 和各阶段 subagent，执行具体检查、审查和文案生成。

  • 生成 .opencode/agents/agentmatrix_driver.md 作为主控 agent 模板。
  • 每个阶段有 executor agent 和 verifier agent 两类角色。
  • subagent 只写 stage-report、evidence 和日志，不拥有工作流状态机。
  • AgentMatrix 校验证据后再决定阶段成功、失败、跳过或继续。
稳定契约是关键：CLI 产生 Stage Invocation，Agent 框架执行并写回证据，AgentMatrix 再验证并更新 run.json。 后续接入 Codex、Claude Code 等平台时，可以替换执行平面，workflow、run state、evidence 和 visualization 仍保持一致。

能力边界与演进方向

AgentMatrix 的定位是交付前流程编排与证据系统，不替代 CI 或代码托管平台。

核心说明

  • AI 写完代码不是结束，真正难点是交付前的验证、审查、修复和复盘。
  • AgentMatrix 把这些步骤显式化，让每个阶段都有状态、输入、输出和证据。
  • 平台 agent 可以变化，核心 workflow 和 run state 保持稳定。
  • 失败不会被口头解释覆盖，而是落到结构化 blocker、命令失败或 verifier 拒绝。
  • 最终产物服务人工合并决策：MR 文案、验证摘要、风险发现和运行可视化。

当前 MVP 边界

  • 不自动创建 GitHub PR 或 GitLab MR。
  • 不执行 git push、分配 reviewer、打 label 或监听 CI。
  • 默认测试套件离线、确定性，不依赖真实模型质量。
  • OpenCode 集成已有模板和协议，但交互式智能执行仍由平台 agent 承担。
  • 后续扩展重点是更多 workflow、更多平台 adapter 和更强的可视化体验。
